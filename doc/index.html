Developer instructions

Overview

This is a GUI tool for running and analyzing clustering algorithms.

Terminology

In literature, the terminology for clustering problems varies, depending mostly on the application. At the University of Joensuu, the following terms are used:

Training set
The data to be partitioned
also known as Data set, Feature vectors, Input data
Codebook
The set of representative vectors for each partition
also known as Centroids, Models, Representative vectors
Partitioning
The mapping of each training vector to a partition

How to use

Opening files

Select Open from the menu or toolbar to open training set, codebooks or partitioning files. You can also drag the files from your file manager to the file list or to the visualization to open them. Files can be in the binary formats used by SIPU software, or you can use plain text files (one vector per line, separated by whitespace).

Note that partitionings are tied to a particular training set. You need to have a compatible training set displayed in the data view to open a partitioning file. After opening, you can select (double-click) a training set; all its partitionings will be marked with a “link” icon.

Displaying files

Double-click an entry in the file list to load it in the data view. Currently, only one training set, one codebook and one partitioning may be displayed at once, so selecting a file hides the previous dataset of the same type.

The results of a process are displayed automatically when the process animation is played.

Double-click a section heading in the file list (“Training sets”, etc.) to remove the corresponding data from the view. (Better UI is needed here)

When a partitioning file is displayed, the corresponding training set is also displayed. Conversely, when a training set is removed from the view, the partitioning is removed as well.

Running clustering processes

Select an algorithm from the list at the bottom. This takes you to a page where you can input paremeters. Most algorithms will allow take training sets and initial codebook as parameters, you can drag files from the file list to set these. Other parameters are set in an intuitive way. You can also specify how the visualization will be animated.

When you are done, press the Run! button to start the process. A movie-player-like interface is provided for viewing results of individual iterations.

Saving results

You can select a file (single-click) and choose Save As to save it.

Select the Save Image action to save a picture of data displayed in the visualization window to an image file. Results are saved in the PNG format by default.

File icons

Several icons can appear next to a file name. Here are their meanings:

Eye
The dataset is currently displayed in the data view
Floppy disk
The dataset is not saved
Link
The partitioning belongs to the currently displayed training set
